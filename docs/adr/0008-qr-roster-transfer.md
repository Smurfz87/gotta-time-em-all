# QR code roster transfer using jsqr

Roster sharing encodes participant names as `{ v: 1, names: [] }` JSON in a QR code generated by the
`qrcode` npm package. Scanning uses `jsqr` processing video frames from `getUserMedia`, polling at 200ms
intervals. The native `BarcodeDetector` API was considered but rejected.

We chose jsqr because `BarcodeDetector` has no support in Safari (iOS or macOS) as of mid-2026, which
covers a significant share of mobile trainers. jsqr is a pure-JS library that works in all browsers and
produces consistent results. The payload is names only — no IDs, no history — keeping QR codes small
(well under 2KB for any realistic roster) and avoiding privacy leakage if a QR code is photographed.

The trade-off is a ~30KB bundle addition (jsqr) and manual frame-capture logic instead of a browser
primitive. If `BarcodeDetector` achieves broad support in the future, the scan path can be swapped out
without changing the payload format or UI.
